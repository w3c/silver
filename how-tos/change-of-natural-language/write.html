Develop

  1. If you’re writing and publishing your own content, you need to mark up any blocks of content in languages different from the language of the page. This might be achieved by using functionality in a text editor’s toolbar, or it might require directly editing code.
  2. If you’re writing content for others to publish, you need to highlight relevant changes of language so that the person who will be adding your content to a site knows how to mark it up.

How

  1. Highlight content using an application’s commenting or reviewing functionality. For example: in a document written in English with a block of content written in French, select the block of French content and add a comment saying “mark up as French”.

Editor Tips

  • Use a text editor that supports marking up or highlighting changes of languages.
  • There isn’t a specific proofreader’s mark for marking up a change of language, but the s/b (“should be”) abbreviation followed by a comment such as “mark up as French” could be used.
