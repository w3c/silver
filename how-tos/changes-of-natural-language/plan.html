Plan

Planning responsibilities

  • Blocks of text that change (human) language are coded so that the language change is communicated to screen readers and other assistive technologies. You must ensure that the author or editor passes the information on a language change to the developers so that the developers code it correctly.
  • If you are building a feature where users can input paragraphs of text that will later be published (like a comment feature or social media post), then you must ensure that the specifications of the project include the ability for users to indicate a change of language. You must then ensure that the teams are working together to implement this feature.

Tips for collaboration

Communication is important with designers, developers, testers in these ways:

  1. If the author or editor is producing content where blocks of human language change (for example, from English to French and back), then they communicate those language changes to the developers so the language change can be coded with the correct language.
  2. Don’t assume that the developer can identify the correct language by reading it.
  3. Ensure that your developer team knows how to code the change of language correctly and that they are referencing the correct IANA language sub-codes. The W3C has a tool for searching Language Subtag Lookup.

  4. If this is a project that includes the ability for users to enter text that is then published (like comments or social media posts), then users need the ability to indicate a change of language. Ensure that:

    1. the designer is involved in developing the user interface for the feature; and
    2. the developer is coding the interface for changing the language in an accessible way that conforms to WCAG.

    Be aware that allowing users to embed code can have security implications.

  5. A site or product with extensive Internationalization, for example, the ability to translate the entire site or product into a selectable language, will need a thorough ability to code the language change everywhere it occurs whenever it occurs. It will need to be as persistent as the words themselves. For example, the code identifying the language as French identifies the French translation. If a page refresh preserves the selected language (French in this example) then the code identifying the French language is also preserved. In this case, you will want to ensure there are no instances where the language change is hard coded and doesn’t flex with the default language.

Planning for each stage

How to get started early

For content that may include language changes, contact the editor or editing team and work out a mechanism for how any change of language needs to be communicated to the development team. This is a communication issue, not a technology issue. The developer needs to know

  1. Where the change of language starts
  2. Where the language change ends, or reverts to the default language
  3. What the languages are. If the language has a specific dialect, then the dialect subcode should be used. For example, Chinese could be Mandarin Chinese (cmn) or Jinyu Chinese (cjy). Historic languages have their own subcodes. For example, quotations from Chaucer’s Canterbury Tales use the subcode for Middle English (enm)

If you are building a new feature that allows users to enter text that will be then published, then you need to start early to determine the features that will be needed to ensure that the text that users enter will be accessible. Language change is a small part of what will be needed.

How to remediate

For an existing feature that allows users to enter text that will then be published, there are a number of considerations. Language change is one of them. Consider using an accessible in-line text editor that can be embedded in your product. If security or functionality limitations require you to build your own text editor with a language change, ensure:

  • The feature can be found by people entering text
  • People can operate the feature with no training or help documentation
  • The feature meets WCAG 3
